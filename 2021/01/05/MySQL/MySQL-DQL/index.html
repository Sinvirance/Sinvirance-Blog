MySQL DQL

发表于 2021-01-05 分类于 MySQL

DQL：Data Query Language 数据查询语言

1.基础查询

  • 查询列表可以是表中的字段、函数、常量值、表达式
  • 查询的结果是一个虚拟表格，不会被保存
起别名

AS 或者 空格

去重

DISTINCT 查询某个字段的非重复记录

2.条件查询

2.1 按条件表达式

> < = != <> >= <=

2.2 按逻辑表达式

|| or && and ! not

2.3 模糊查询
2.3.1 [NOT] LIKE

常见通配符

  • %：任意长度字符串 包含0个

  • _：任意一个字符串

  • [NOT] BETWEEN...AND...：范围查询，包含临界值

2.3.2 [NOT] IN

查询包含给定集合中的字段数据，和 or效果类似，但是集合中的数据类型必须一致或者兼容

2.3.3 IS [NOT] NULL

3.排序查询

  • ASC：从低到高，不写默认升序

  • DESC：从高到低

  • 排序列表可以是表字段、表达式、函数等

  • 可以使用多个字段排序

4.分组查询

将表中的数据分成若干组

注：在使用group by 时，select中的字段不可以单独出现，必须出现在 group语句中或者在函数中

  • having： 分组查询后对结果过滤

5.连接查询

5.1 笛卡尔积

​ 无任何连接条件，查询的结果条数为两个表记录相乘，有重复数据\

5.2 内连接

​ 查询两个表符合连接条件的记录，也就是查交集

5.2.1 等值连接
5.2.2 非等值连接
5.2.3 自连接
5.3 外连接

​ 主表和从表公有的部分正常显示，主表想要去从表查找的记录查找不到的，以null填充

5.3.1 左外连接

查询左表独有部分

5.3.2 右外连接

查询右表独有部分

5.3.3 全外连接

查询左右表全部的内容，将左右表重复的只保留一份

查询左右表全部的内容，将左右表公共部分删除

6.子查询

内层语句的查询结果可以作为外层语句的条件

6.1 IN 子查询:

可以帮助我们查询两个表之间的脏数据

6.2 比较运算符子查询
6.3 EXISTS 子查询:

EXISTE 后面是一个返回值是 boolean 类型的判断语句

6.4 ANY SOME ALL
运算符 ANY SOME ALL
>、>= 最小值 最小值 最大值
<、<= 最大值 最小值 最小值
= 任意值 任意值
<>、！= 任意值
6.5 子查询创建或者更新表

7.分页查询

  • offset：起始索引，从0开始
  • size：从起始索引开始查询的条数

8.联合查询

UNION

去掉两个表中重复值,自己表中重复的不管

UNION ALL

简单合并,不去除重复值
