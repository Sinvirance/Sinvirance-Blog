1. Log4j2 简介

Apache Log4j 2是日志框架Log4j的升级，它比其前身Log4j 1.x提供了重要的改进，并且参考了Logback中许多有用的改进，同时修复了Logback的一些固有问题。

– Apach Log4j2 官方

2. Log4j2 与 Log4j 的对比

  • 配置文件类型

    Log4j 主配置文件是 log4j.properties 文件，Log4j2 主配置文件是 log4j2.xml、log4j2.json 或者log4j2.jsn

  • 引入的 Jar 文件不同

    • 性能对比：Log4j2 具有同步日志和异步日志两种日志模式，在高并发下性能远高于 Log4j

3. Log4j2 常见概念

  • 日志级别： log4j2 自带有8种日志级别，高->低：OFF、FATAL、ERROR、WARN、INFO、DEBUG、TRACE、ALL

HDFS 分布式文件系统

1. HDFS 简介

The Hadoop Distributed File System (HDFS) is a distributed file system designed to run on commodity hardware. It has many similarities with existing distributed file systems. However, the differences from other distributed file systems are significant. HDFS is highly fault-tolerant and is designed to be deployed on low-cost hardware. HDFS provides high throughput access to application data and is suitable for applications that have large data sets. HDFS relaxes a few POSIX requirements to enable streaming access to file system data. HDFS was originally built as infrastructure for the Apache Nutch web search engine project. HDFS is part of the Apache Hadoop Core project.

  • Distributed： 分布式

  • run on commodity hardware: 运行在通用硬件

  • highly fault-tolerant: 高容错

  • highly throughput: 高吞吐

Zookeeper 介绍

发表于 2021-05-20 分类于 Zookeeper

Zookeeper 介绍

1. Zookeeper 是什么

Zookeeper 是一个 分布式协调服务的开源框架。主要用来解决分布式集群中应用系统的 数据一致性问题，例如避免同时操作同一数据造成的脏读问题。

  • Zookeeper 本质上是一个分布式的小文件存储系统：提供类似文件系统的目录树方式的数据存储，并且对树节点进行有效管理
  • Zookeeper 提供给客户端监控存储在 Zookeeper 服务器内部数据变化的功能：从而可以达到基于数据的集群管理。例如： 注册中心服务（dubbo）、分布式配置管理（solr的配置集中管理）、分布式消息队列（sub/pub）、分布式锁、分布式协调等功能

2. Zookeeper 架构组成

​

Leader

  • Zookeeper 集群工作的 核心角色
  • Zookeeper 集群中各个服务器的调度者
  • Zookeeper 中事务请求（Write）操作的唯一调度和处理者
  • Leader 不由用户决定，而是由 Follewer 选举出来

Follower

  • 处理客户端非事务的请求（Read）
  • 转发事务请求（Write）给 Leader
  • 参与集群中 Leader 的选举投票

Observer

  • Zookeeper 集群工作中的 观察者角色：观察集群中的最新变化并且将变化状态同步
  • 处理客户端非事务的请求，将事务请求转发给 Leader
  • 不参与任何形式的投票，只是用来提高集群的非事务处理能力，增加集群并发的读请求

Zookeeper 也是 Master/Slave 架构，与一般的不同的是，Leader不是用户指定而来，而是通过 Follower 选举投票产生

3. Zookeeper 特点

  1. 一个 Leader，多个 Follower 组成的集群
  2. Leader 负责进行各种投票的发起与决议，更新系统状态
  3. Follower 用于接收客户请求并向客户端返回结果，在选举 Leader 过程中参与投票
  4. Zookeeper 集群中只要有半数以上节点存活，就能正常服务
  5. 全局数据一致性：每个服务器（Server） 都保存一份相同的数据副本，客户端（Client）无论连接到哪个Server，数据都是一致的
  6. 更新请求顺序进行
  7. 数据更新是原子性，一次数据更新要么全部成功，要么全部失败
