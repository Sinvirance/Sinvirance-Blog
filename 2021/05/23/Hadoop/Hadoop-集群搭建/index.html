发表于 2021-05-23 更新于 2021-07-03 分类于 Hadoop

Hadoop 集群搭建

单机模式：单节点模式，非集群

单机伪分布式：单节点，多线程模拟集群的效果

完全分布式模式：多台节点，真正的分布式 Hadoop 集群搭建，生产环境使用方式

1. 虚拟机环境准备

  • 3台虚拟机(静态ip，防火墙关闭，集群时间同步，ssh免密登录，修改主机名)

  • 创建软件安装文件夹

  • jdk 环境变量配置

  • Hadoop 2.9.2 下载地址：https://archive.apache.org/dist/hadoop/common/hadoop-2.9.2/hadoop-2.9.2.tar.gz

2. 集群节点规划

框架 centos100 centos101 centos102
HDFS NameNode、DataNode DataNode SecondaryNameNode、DataNode
YARN NodeManager NodeManager ResourceManager、NodeManager

3. 安装 Hadoop

  • 解压 Hadoop

  • 添加 Hadoop 到环境变量配置文件，并应用配置文件

  • 验证 Hadoop

  • Hadoop 目录结构

    3.1 集群配置

修改 Hadoop 配置目录文件：/opt/sin/servers/hadoop-2.9.2/etc/hadoop

HDFS 集群配置

  • 将 JDK 路径明确配置给 HDFS（修改 hadoop-env.sh）

  • 指定 NameNode 节点以及数据存储目录（修改 core-site.xml）

  • 指定 SecondaryNameNode 节点（修改 hdfs-site.xml）

  • 指定 DataNode 从节点（修改 /etc/hadoop/slaves 文件）

    MapReduce 集群配置

  • 将 JDK 路径明确配置给 MapReduce（修改 mapred-env.sh）

  • 指定 MapReduce 计算框架运行 Yarn 资源调度框架（修改 mapred-site.xml）

    Yarn 集群配置

  • 将 JDK 路径明确配置给 yarn（修改 yarn-env.sh）

  • 指定 ResourceManager 节点所在的计算机节点（修改 yarn-site.xml）

  • 指定 NodeManager 节点，与DataNode 共用配置（ /etc/hadoop/slaves 文件）

注意

Hadoop 安装目录默认是 501 dialout，我们操作虚拟机用户是root，避免出现信息紊乱，修改 Hadoop 的用户和用户组

3.2 分发配置

rsync 介绍

  • rsync：Linux 远程文件同步应用工具

    特点：rsync 会检查发送方和接收方已有的文件，仅传输有变动的部分（默认规则是文件大小或修改时间有变动）

    注意：传输的双方都必须安装 rsync

  • 安装

  • 基本语法

  • 选项参数说明

    选项 功能
    -r（必须参数） 递归
    -v 显示复制过程
    -l 拷贝符号连接
    -a（可以代替-r） 递归，同步元信息（比如修改时间、权限等）
rsync-script 集群分发脚本编写

  • 脚本实现需求

    1. 循环复制文件到集群所有节点的相同目录下
    2. 使用 rsync-script + 路径(绝对路径或者相对路径) 来执行脚本
    3. root 用户可以在系统任意地方直接执行，通过存放脚本在 /usr/local/bin

  • 脚本具体实现

    1. 在 /usr/local/bin 目录下创建文件 rsync-script
